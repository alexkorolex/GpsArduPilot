
## Быстрый запуск

Откройте терминал в папке `release`:

```sh
cd release
chmod +x ./arduplane
./arduplane \
  -w \
  --model plane \
  --speedup 1 \
  --slave 0 \
  --sim-address=127.0.0.1 \
  -I0 \
  --home 55.7522,37.6156,180,0
```

`--model plane` обязателен: он выбирает симулятор самолета.

`--home` задает стартовую точку в формате:

```text
широта,долгота,высота_м,курс_градусы
```

`-w` сбрасывает сохраненные параметры перед стартом. Если нужно сохранить
настройки между запусками, уберите `-w`.

## Подключение MAVProxy и QGroundControl

Сам бинарь запускает ArduPlane SITL. Для консоли MAVProxy и подключения
QGroundControl запустите второй терминал:

```sh
mavproxy.py \
  --retries 5 \
  --master tcp:127.0.0.1:5760 \
  --out 127.0.0.1:14550 \
  --sitl 127.0.0.1:5501
```

QGroundControl обычно подключается автоматически по UDP `127.0.0.1:14550`.
Если автоматического подключения нет, добавьте UDP-подключение вручную на порт
`14550`.

Для запуска самого `arduplane` Python не нужен. Python/MAVProxy нужен только
для консоли MAVProxy и UDP-моста в QGroundControl.

## Включение inertial/no-GPS режима

После подключения через MAVProxy выполните:

```text
param set AHRS_EKF_TYPE 3
param set EK3_ENABLE 1
param set EK3_PLN_GPS 0
reboot
```

После `reboot` не запускайте следующий старт с `-w`, иначе сохраненные
параметры будут сброшены.

Вернуть обычный режим можно так:

```text
param set EK3_PLN_GPS 1
reboot
```

## Остановка и файлы состояния

Остановить SITL и MAVProxy можно через `Ctrl+C` в соответствующих терминалах.

При запуске в текущей папке могут появляться файлы состояния, например
`eeprom.bin`. Это нормально: там хранятся параметры симулятора.

## Частые проблемы

Если терминал пишет `permission denied`, выполните:

```sh
chmod +x ./arduplane
```

Если терминал пишет `bad CPU type in executable` или файл не запускается на
другой ОС, нужен отдельный бинарь под целевую платформу.

Если MAVProxy не подключается, проверьте, что `arduplane` уже запущен и порт
`5760` не занят другим процессом.
